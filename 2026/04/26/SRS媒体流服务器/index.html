SRS媒体流服务器

发表于 2026-04-26

简介

SRS（Simple Realtime Server）是一款开源实时媒体服务器，适配面广、上手成本低，在直播与 RTC 场景里应用广泛，社区与文档都比较成熟。

配置要点说明

一、rtc_server（全局 RTC）

项 含义
enabled 是否开启 RTC 服务器，默认 off。
listen RTC 侦听端口（UDP）。
candidate 对外宣告的候选地址；WebRTC 场景下通常必须配置正确。
tcp.listen 若启用 TCP 传媒体，为 WebRTC over TCP 的侦听端口。

二、各 vhost 内的 rtc 块

项 含义
rtc.enabled 该 vhost 是否启用 RTC，默认 off。
rtc.rtmp_to_rtc 是否 RTMP 转 RTC。
rtc.rtc_to_rtmp 是否 RTC 转 RTMP。
rtc.stun_timeout 会话超时（秒）。
rtc.nack 是否 NACK（丢包重传），默认 on。
rtc.twcc 是否 TWCC（拥塞反馈），默认 on。
rtc.dtls_role DTLS 角色：active 为客户端主动，passive 为服务端被动。

云服务器部署与安全

  • 反向代理：可将 8080、1985 走反向代理，用固定域名提供 WebRTC 相关能力，减少直接暴露公网 IP；8000/UDP 一般仍需在安全组/防火墙放行。
  • HTTP API 回调：SRS 支持推流等事件的 HTTP 回调，可在回调里校验 token（例如带时效的临时密钥），由业务后端决定是否允许推流，从而加强直播链路安全。
  • 白名单：可配置推流/播放 IP 白名单，非名单内 IP 拒绝推流或播放。

注意事项

部分场景下推流端只有视频、没有音频，拉流端需做兼容处理；否则可能在 DTLS 握手阶段出现异常，导致拉流失败。
